# Risk Register — Feature 068 Search and EvidenceBundle API v1

| ID | Risk | Control implemented | Residual risk / required next gate |
|---|---|---|---|
| SE-01 | Cross-tenant retrieval | Forced RLS, explicit tenant predicates, transaction-local tenant context, tenant/request/credential binding, cross-tenant smoke | Consumer contract tests and staging multi-tenant tests remain pending. |
| SE-02 | Private object or scanner coordinates leak through retrieval | Column-level runtime grants; response projection excludes object namespace/key/version, scanner engine/version, lease/fencing tokens and pipeline config | Production privilege drift monitoring and periodic privilege review remain pending. |
| SE-03 | Client promotes a source to official/current/supported | Closed schemas reject authority state; classifications derive from validated source state and stored dates | Human review of source validation and legal supersession remains pending. |
| SE-04 | Semantic/hybrid silently degrades or holds database work during provider latency | Capability is prepared before PostgreSQL retrieval, the HTTP transport has a bounded abort timeout, and missing/timed-out/failing provider returns `503` | Provider SLOs, retry/circuit-breaker policy, cost limits and incident drills remain pending. |
| SE-05 | Duplicate artifacts/jobs produce duplicate evidence | Citation-identity deduplication keeps the strongest ranked candidate | Explain-plan and high-cardinality corpus tests remain pending. |
| SE-06 | Negative cosine similarity violates the response contract | Similarity is clamped at zero and score semantics are explicit | Corpus-specific calibration and threshold evaluation remain pending. |
| SE-07 | Comparative evidence becomes an authoritative claim | EvidenceBundle ordinary claims require `supported`; comparative references remain citations/gaps with Antigua corroboration warning | Human review must still verify classification and context. |
| SE-08 | Conflicting versions are silently resolved | Conflicts create review-required positions, contradiction and gap; affected ordinary claims are removed | Automated conflict detection is location/text based and does not replace semantic review. |
| SE-09 | Hash-valid replay is semantically corrupted | Replay validates schema, hash, request/provenance identity, source/citation relationships, exact claim excerpts, authority/support constraints, deterministic IDs and contradiction references; invalid state is deleted in a committed transaction | Database compromise remains outside application guarantees; key management and DB auditing are production gates. |
| SE-10 | Malformed legacy digest poisons all retrieval | SQL compares encoded accepted digest to text SHA rather than decoding untrusted legacy text; malformed rows fail closed | Data repair and integrity scans for legacy rows remain pending. |
| SE-11 | Signed or credential-bearing URL leaks | Database constraints plus application public-URL validation reject credentials and common signature/token parameters | Allowlist governance, redirect inspection and URL liveness scanning remain pending. |
| SE-12 | Rate/idempotency tables grow without bound | Replay entries expire logically and claim cleanup removes expired matching scope; rate state is bounded per row | Scheduled retention cleanup, metrics and storage alerts remain pending. |
| SE-13 | Local green gate is mistaken for real retrieval quality | Documentation and responses state that fixtures do not prove corpus completeness, relevance, authority, vigencia or legal applicability | Antigua-first acquisition, real-corpus keyword/phrase/semantic/hybrid evals and human citation review remain blocking. |
| SE-14 | Feature is presented as deployed production capability | Program status distinguishes local verified branch from merge/deploy; no deployment action is performed | Reviewed PR, protected merge, staging and observed deployment remain pending human/operational gates. |
