# LA Muni RAG — Guía de gobernanza para presentación municipal

## Principio de uso

LA Muni RAG debe presentarse como una herramienta de consulta documental con evidencia. Su función es ayudar a encontrar, resumir y revisar documentos municipales; no sustituye interpretación legal, dictamen técnico ni decisión de autoridad competente.

## Promesa aceptable

El sistema puede:

- buscar en documentos municipales cargados al corpus;
- responder en español con una síntesis breve;
- mostrar fuentes verificadas;
- indicar cuando la evidencia es limitada;
- permitir revisión de fragmentos y páginas;
- explicar de forma visual la ruta de recuperación en Glass Wall.

El sistema no debe prometer:

- criterio legal final;
- decisión administrativa automática;
- cobertura completa fuera del corpus cargado;
- exactitud absoluta sin revisión humana;
- sustitución de procedimientos municipales.

## Política de evidencia

Una respuesta debe considerarse presentable cuando incluye:

1. síntesis clara;
2. hallazgos principales;
3. fuentes verificadas visibles;
4. página o referencia documental cuando esté disponible;
5. estado de cobertura o evidencia;
6. posibilidad de revisar el fragmento.

Si la evidencia es insuficiente, la respuesta debe decirlo de forma explícita y no debe inventar conclusiones.

## Lenguaje recomendado

Usar:

- Evidencia sólida
- Evidencia suficiente
- Evidencia limitada
- Fuentes verificadas
- Cobertura documental
- Revisión humana recomendada

Evitar en presentación pública:

- confianza baja;
- respuesta definitiva;
- el sistema decidió;
- garantía total;
- términos internos de implementación.

## Revisión humana

Para uso institucional, se recomienda que respuestas usadas en actas, informes, comunicación pública o decisiones administrativas pasen por revisión de:

- unidad técnica correspondiente;
- asesoría legal si aplica;
- comunicación institucional si será publicado;
- autoridad competente si será parte de decisión formal.

## Criterios de corpus

Antes de una demo oficial, confirmar:

- qué documentos están cargados;
- fecha aproximada de actualización;
- si los documentos son públicos;
- si hay versiones vencidas o borradores;
- si las páginas y metadatos están bien normalizados;
- si existen documentos sensibles que no deben estar disponibles.

## Manejo de objeciones

### Legal

Respuesta recomendada:

> El sistema no emite dictamen. Presenta evidencia documental para revisión humana.

### IT

Respuesta recomendada:

> La interfaz pública no expone detalles operativos sensibles. Glass Wall muestra una ruta sanitizada de consulta y evidencia.

### Comunicación

Respuesta recomendada:

> El lenguaje se mantiene ciudadano e institucional. Las fuentes están visibles para evitar respuestas opacas.

### Concejo o alcaldía

Respuesta recomendada:

> Esto ayuda a preparar conversaciones mejor fundamentadas, no a sustituir deliberación ni decisión municipal.

## Señales de listo para presentar

- La homepage se entiende sin explicación técnica.
- El widget responde con síntesis y fuentes visibles.
- Las evidencias se leen como documentos municipales, no como logs técnicos.
- El estado de evidencia es prudente y no alarmista.
- Glass Wall explica el flujo sin filtrar datos sensibles.
- El equipo puede responder preguntas sobre alcance, límites y revisión humana.

## Recomendación final

Para una municipalidad, la confianza no viene de decir que la IA sabe más. Viene de demostrar que cada respuesta tiene fuente, límite, contexto y una ruta clara para revisión humana.
